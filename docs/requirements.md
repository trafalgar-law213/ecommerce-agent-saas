# 项目需求文档

> 最后更新：2026-07-03
> 本文档描述项目的业务目标和功能需求，用通俗语言编写，适合所有角色阅读。

---

## 一、项目背景

中小电商团队在日常运营中面临三个痛点：

1. **数据看不懂**：有销售 CSV 但不会写 SQL/Python 分析，不知道哪些商品赚钱、哪些滞销
2. **文案写不好**：运营人手有限，写不出多种风格的营销文案（朋友圈、小红书、详情页）
3. **SOP 记不住**：公司有退货流程、客服话术等文档，但新员工找不到、老员工记混

本项目的目标：做一个 **AI 助手网站**，让电商运营人员用**自然语言**就能完成以上三件事。

---

## 二、目标用户

| 用户角色 | 使用场景 |
|----------|----------|
| 电商运营 | 上传销售 CSV，问"Top 5 商品是什么" |
| 客服主管 | 上传 SOP 文档，问"7 天无理由退货怎么处理" |
| 市场营销 | 让 AI 生成商品的朋友圈/小红书推广文案 |
| 老板/经理 | 查看整体销售情况，不用打开 Excel |

---

## 三、核心功能（MVP）

### 功能 1：多轮对话助手

- 用户在一个类似聊天软件的界面上输入问题
- AI 能记住上下文（比如先问"昨天的销量"，再问"那前天呢？"AI 知道在说销量）
- 对话历史被保存，刷新页面后不丢失
- 可以新建/切换/删除会话

### 功能 2：销售数据上传与分析

- 支持上传 CSV 文件（电商平台导出的销售报表）
- AI 自动理解列名（中文列名也支持）
- 支持的提问类型：
  - "帮我分析 Top 10 商品"
  - "计算每个分类的利润率"
  - "最近 30 天的销售趋势"
  - "哪个商品销量最高但利润最低"
- 返回结果包含表格 + 图表建议

### 功能 3：知识库管理（RAG）

- 支持上传 PDF、Word、TXT 文档
- 后台自动把文档"切片"并存入向量数据库
- 用户提问时，AI 从文档中检索相关片段，引用原文回答
- 示例："退货流程是什么？" → AI 引用 SOP 文档中的退货章节

### 功能 4：营销文案生成

- 根据商品信息生成文案
- 支持多种风格：活泼（小红书）、专业（详情页）、简洁（朋友圈）
- 示例："帮我给这款防晒霜写一段小红书风格的种草文案"

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| 可用性 | `docker-compose up` 一条命令启动 |
| 部署 | 有公网可访问的演示地址 |
| 代码 | 模块化、有注释、有 README |
| Git | 公开仓库，提交历史清晰 |
| 文档 | 有架构图、快速开始、演示 GIF |
| 兼容性 | Windows/Mac/Linux 均可运行 |

---

## 五、不做的事（MVP 范围外）

- 用户注册/登录（第 1 版不需要）
- 多租户数据隔离
- 支付/收费
- 实时协作
- 移动端 App
